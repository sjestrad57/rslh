Overview
What this app does and how it works

STR Listing Configurator automatically enforces tier-based listing limits for short-term rental businesses running on GoHighLevel's Rental module.

When a client reaches their plan limit and attempts to add a new listing, a branded upgrade prompt fires automatically — stopping the action and presenting a clear call to upgrade their plan.

💡 No hard errors, no broken experience. Just a clean, professional soft block that turns every limit hit into an automatic upsell opportunity.
1
Install the App
Install on your agency account through GHL Marketplace
  • ✓
    Go to your GHL Agency Dashboard
  • ✓
    Navigate to App Marketplace
  • ✓
    Search for STR Listing Configurator
  • ✓
    Click Install — the app is now active on your agency
🔒 This is a private, agency-only app. Only your agency can install and manage it across client sub-accounts.
2
Create Custom Fields
Set up two custom fields in each client sub-account

Navigate to the client sub-account and create these two fields:

Sub-account Settings → Custom Fields → + Add Field
rental_tier
Type: Dropdown (single) Object: Contact Options: starter, growth, pro
listing_limit
Type: Number Object: Contact Example values: 5, 20, 999
📌 Field keys will be assigned automatically as contact.rental_tier and contact.listing_limit. The app reads these keys to enforce limits.
3
Set Client Tier
Configure each client's plan limits at onboarding

For each new client, set their tier values in their sub-account custom fields:

  • ✓
    Open the client's sub-account
  • ✓
    Go to Settings → Custom Fields
  • ✓
    Set rental_tier to their plan (starter, growth, or pro)
  • ✓
    Set listing_limit to their allowed number of listings
Tier rental_tier value listing_limit value Notes
Starter starter 5 Entry level plan
Growth growth 20 Mid-tier plan
Pro pro 999 Use 999 for unlimited
4
Go Live — Client Experience
What your client sees once configured

Once configured, the app runs automatically with zero maintenance:

  • ✓
    Client logs into their sub-account via app.accompels.com
  • ✓
    Navigates to Calendars → Rentals → Listings
  • ✓
    Adds listings freely up to their plan limit
  • ✓
    At the limit — clicks + New Listing
  • ✓
    Branded upgrade modal fires automatically
  • ✓
    Client clicks Upgrade My Plan → directed to your pricing page
⚡ Every limit hit is an automatic upsell. No manual follow-up needed — the app handles the upgrade prompt for you.
5
Managing Upgrades
What to do when a client upgrades their plan

When a client upgrades their plan, update their custom fields:

  • ✓
    Go to the client's sub-account
  • ✓
    Navigate to Settings → Custom Fields
  • ✓
    Update rental_tier to their new tier
  • ✓
    Update listing_limit to their new limit
  • ✓
    Client can immediately add more listings — no restart needed
🔄 Phase 2 Enhancement: Connect Stripe webhooks to GHL workflows to auto-update custom fields when a client upgrades — fully automated with zero manual steps.

Troubleshooting
Common issues and how to resolve them
❓ Modal not firing when client clicks + New Listing
Confirm the client is accessing GHL through app.accompels.com and not app.gohighlevel.com. Verify listing_limit custom field is set correctly and the client is at or above their limit.
❓ Listing count appears incorrect
Refresh the Rentals listings page and try again. If the issue persists, contact support — the listing count selector may need updating due to a GHL platform update.
❓ Client can still add listings after reaching limit
Verify the custom fields are set at the sub-account level and not the agency level. Also confirm the client is using a User-level role and not an Admin role.
❓ Modal shows wrong plan name or limit number
Check that rental_tier and listing_limit custom field keys match exactly: contact.rental_tier and contact.listing_limit. Field keys are case-sensitive.
